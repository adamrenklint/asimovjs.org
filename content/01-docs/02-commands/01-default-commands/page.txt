The <code>ajs</code> command line interface comes equiped with a few powerful default commands, allowing you to run, test and manage your entire project and it's pages - directly from your terminal window.

## ajs help

Show usage instructions, basically a shorter version of this page.

## ajs create [name]

Create a new project from the [default project template](https://github.com/adamrenklint/asimovjs-template-project).

## ajs / ajs start

Start your asimov.js project, equivalent to running <code>node main.js</code>. Optionally pass <code>--open</code> to open the project in your default browser.

## ajs new page [url] [template]

Create a new page in the <code>/content</code> folder.

For example, <code>ajs new page /blog/01-hello-world post</code> will create a new page in <code>/content/blog/01-hello-world/post.txt</code>.

## ajs new template [name]

Create a blank template in <code>/site/templates</code>.

## ajs new style [name]

Create a blank stylesheet in <code>/site/styles</code>.

## ajs extend [parent] [path]

Create subclass of <code>parent</code> in <code>path</code>. Finds parent class by searching for first match in the <code>/lib</code> folder.

asimov.js also exposes a few base classes: Base, Initializer, Middleware, Helper and Command.

The extended class will by default be created in the same folder as the parent class, unless you define a folder path.

For example: <code>ajs extend MyCustomClass newFolder/AwesomerClass</code> will create <code>AwesomerClass.js</code> in <code>/lib/newFolder</code>, regardless of where the parent is saved.

## ajs debug

Start project with verbose logging. The <code>--open</code> flag also works here.

## ajs test

Run all tests in the <code>/tests</code> folder.

To use any of mocha's built-in reporters, set the <code>-R</code> flag. For example, use <code>ajs test -R spec</code> to use the spec reporter.

## ajs test [grep]

Run only the tests matching <code>grep</code>. The <code>-R</code> flag of course also works here.

## ajs coverage

Generate a test coverage report using Istanbul. Pass the <code>--open</code> flag to open the report in your default browser.

## ajs publish

Publish project to NPM and push git tag. Used by asimov.js internally to publish itself.

To publish a beta version of your project ot NPM, use the <code>--beta</code> flag. This will make it installable to users with <code>npm install yourModule@beta</code>, but not be considered when doing a normal <code>npm install yourModule</code>.
